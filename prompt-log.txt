Models used:
- ChatGPT (for planning)
- Claude code (for coding)
**This was the first time I tried using Claude code connected to my Github to create my project. Usually I use the Claude web browser and copy everything over to VScode manually for better testing. However, I decided to try something new this time. It took some time to properly setup Claude code, but I am pleased with the overall result and workflow. It was highly efficient compared to manually copy pasting. However, I noticed that you need to give it very specific prompts when you get to late stage project implementations. If you ask it to fix the bug it might only fix the bug in one page, but skip the other pages since there's so many. 
**Workflow was first planning and iterating on the plan with ChatGPT. Then I fed the plan to Claude code and had it generate each phase. At each phase I told it to give me terminal commands I could run to test the functionality of everything. 

Planning stage prompts to ChatGPT:
- I want to create a budget tracker where you can manually add money you've spent, but also upload your receipts and applepay screenshots and it also adds automatically
- create this into a comprehensive project plan with techstack and architecture with phases that I can feed into claude to let claude build the project
- Add these requirements to the phases: Core Requirements (your app must do all of these) Your app must use a SQLite database with at least one table containing at least 4 columns (including an id column). Your app must support all four CRUD operations: Operation What It Means Example Create Add a new record to the database Add a new book to my reading log Read Retrieve and display records Show all books I've read this year Update Modify an existing record Change a book's rating from 3 to 4 stars Delete Remove a record from the database Remove a book I added by mistake Your Read functionality must include at least one query that filters or sorts the data based on user input (e.g., "show only 5-star books" or "show entries sorted by date"). Graphical interface: A desktop app using a library like tkinter or PyQt. Only attempt this if you're comfortable or curious.
